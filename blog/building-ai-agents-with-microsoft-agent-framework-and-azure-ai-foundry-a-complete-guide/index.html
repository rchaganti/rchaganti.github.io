Introduction to Microsoft Agent Framework Microsoft Agent Framework - Creating Azure AI Foundry Agents

The Microsoft Agent Framework (MAF) is an open-source development kit that combines the best ideas from Semantic Kernel and AutoGen projects. It provides a flexible foundation for building AI agents that can:

  • Process user inputs using Large Language Models (LLMs)
  • Call tools and MCP servers to perform actions
  • Generate intelligent, context-aware responses
  • Manage conversation state across interactions

In this guide, we’ll explore five progressively complex examples that showcase different capabilities of Foundry agents.

Prerequisites

Before diving into the examples, make sure you have:

  1. Python 3.10+ installed
  2. Azure CLI installed and authenticated (az login)
  3. Azure AI Foundry project set up with a deployed model
  4. Required packages installed:
1

pip install agent-framework[azure-ai] azure-identity python-dotenv
  1. Environment variables configured:
1
2

export AZURE_AI_PROJECT_ENDPOINT="https://<your-project>.services.ai.azure.com/api/projects/<project-id>"
export AZURE_AI_FOUNDRY_MODEL_DEPLOYMENT_NAME="gpt-4o-mini"

Let’s start with the simplest possible agent - one that requires minimal setup and demonstrates the core concepts.

import asyncio
from agent_framework.azure import AzureAIAgentClient
from azure.identity.aio import AzureCliCredential

async def main():
    async with (
        AzureCliCredential() as credential,
        AzureAIAgentClient(
            project_endpoint="https://ai-engineer-in.services.ai.azure.com/api/projects/ai-engineer-in",
            model_deployment_name="gpt-5-mini",
            async_credential=credential,
            agent_name="WeatherAgent",
        ).create_agent(
            instructions="You are a weather man. Provide accurate and concise weather information based on user queries.",
        ) as agent,
    ):
        result = await agent.run("What's the weather like in Bengaluru?")
        print(result.text)

asyncio.run(main())

Key Concepts

  1. Async Context Managers: The code uses Python’s async with statement for proper resource management. Both the credential and agent are managed as async context managers, ensuring proper cleanup.

  2. AzureCliCredential: This authenticates using your Azure CLI session - no need to manage API keys in your code or environment variables.

  3. AzureAIAgentClient: The main client for creating Foundry agents. It accepts:

    • project_endpoint: Your Azure AI Foundry project URL
    • model_deployment_name: The deployed model to use
    • async_credential: Your authentication credential
    • agent_name: A human-readable name for your agent

  4. Agent Instructions: The instructions parameter defines the agent’s persona and behavior. This is the system prompt that guides how the agent responds.

  5. Running the Agent: The agent.run() method sends a message and returns the complete response.

Why This Matters

This example shows that creating a basic AI agent requires just a few lines of code. The framework handles all the complexity of API calls, authentication, and response parsing.

import asyncio
from agent_framework.azure import AzureAIAgentClient
from azure.identity.aio import AzureCliCredential

async def main():
    async with (
        AzureCliCredential() as credential,
        AzureAIAgentClient(
            project_endpoint="https://ai-engineer-in.services.ai.azure.com/api/projects/ai-engineer-in",
            model_deployment_name="gpt-5-mini",
            async_credential=credential,
            agent_name="WeatherAgent",
        ).create_agent(
            instructions="You are a weather man. Provide accurate and concise weather information based on user queries.",
        ) as agent,
    ):
        print("Agent: ", end="", flush=True)
        async for chunk in agent.run_stream("Tell me a short story"):
            if chunk.text:
                print(chunk.text, end="", flush=True)
        print()

asyncio.run(main())

Key Differences from Example 1

  1. run_stream() vs run(): Instead of waiting for the complete response, run_stream() returns an async iterator that yields chunks as they arrive.

  2. Chunk Processing: Each chunk may contain text (or may be empty for control messages), so we check if chunk.text before printing.

  3. Real-time Output: Using flush=True ensures each chunk appears immediately in the console, creating a typing effect.

When to Use Streaming

  • Interactive applications: Chat interfaces, command-line tools
  • Long-form content: Stories, explanations, code generation
  • User experience: Provides immediate feedback that the agent is working

import asyncio
from typing import Annotated
from agent_framework.azure import AzureAIAgentClient
from azure.identity.aio import AzureCliCredential
from pydantic import Field
from dotenv import load_dotenv
import os

load_dotenv()

def get_weather(
    location: Annotated[str, Field(description="The location to get the weather for.")],
) -> str:
    import requests
    import json

    api_key = os.getenv("OPEN_WEATHERMAP_API_KEY")
    base_url = "http://api.openweathermap.org/data/2.5/weather"
    complete_url = f"{base_url}?q={location}&appid={api_key}&units=metric"

    response = requests.get(complete_url)
    data = response.json()

    if data["cod"] != "404":
        main_data = data["main"]
        current_temperature = main_data["temp"]
        return f"Temperature: {current_temperature}Â°C"
    else:
        return "City not found"

async def main():
    async with (
        AzureCliCredential() as credential,
        AzureAIAgentClient(async_credential=credential).create_agent(
            name="WeatherAgent",
            instructions="You are a helpful weather assistant.",
            tools=get_weather,
            store=True
        ) as agent,
    ):
        result = await agent.run("Given the current climate in Austin Texas, what should I wear?")
        print(result.text)

asyncio.run(main())

Key Concepts

  1. Typed Tool Functions: The get_weather function uses Python type annotations with Annotated and Pydantic’s Field to describe parameters. This metadata is automatically converted to the function schema that the LLM understands.

  2. Tool Registration: Pass your function(s) to the tools parameter when creating the agent. You can pass a single function or a list of functions.

  3. Automatic Tool Selection: The agent automatically decides when to call your tool based on the user’s query. Ask about weather? It calls get_weather. Ask about something else? It uses its built-in knowledge.

  4. Real API Integration: This example connects to the OpenWeatherMap API to fetch actual weather data - demonstrating how agents can interact with external services.

  5. The store=True Parameter: This enables conversation persistence on the server side.

The Magic of Tool Calling

When you ask “What should I wear in Austin Texas?”, the agent:

  1. Recognizes it needs weather information
  2. Calls get_weather("Austin Texas")
  3. Receives the temperature data
  4. Combines that with its knowledge about clothing to give a contextual recommendation

This pattern is fundamental to building agents that can take real actions in the world.

import asyncio
from agent_framework import HostedCodeInterpreterTool
from agent_framework.azure import AzureAIAgentClient
from azure.identity.aio import AzureCliCredential

from dotenv import load_dotenv  
load_dotenv()

async def main():
    async with (
        AzureCliCredential() as credential,
        AzureAIAgentClient(async_credential=credential).create_agent(
            name="CodingExecutionAgent",
            instructions="You are a helpful assistant that can write and execute Python code.",
            tools=HostedCodeInterpreterTool()
        ) as agent,
    ):
        result = await agent.run("Calculate the 100th prime number.")
        print(result.text)

asyncio.run(main())

Key Concepts

  1. HostedCodeInterpreterTool: A built-in tool that provides a sandboxed Python execution environment hosted in Azure. The agent can write and run code without affecting your local system.

  2. Complex Problem Solving: Ask the agent to calculate the 100th prime number, and it will:

    • Write Python code to find prime numbers
    • Execute the code in the sandbox
    • Return the result (541, by the way!)

  3. Safe Execution: The code runs in an isolated environment, so the agent can experiment without risk.

Use Cases for Code Interpreter

  • Mathematical calculations: Complex formulas, statistical analysis
  • Data processing: Parsing, transforming, analyzing data
  • Code generation and testing: The agent can write code and verify it works
  • Visualization: Generate charts and graphs (when combined with file handling)

import asyncio
import os
from agent_framework import ChatAgent
from agent_framework.azure import AzureAIAgentClient
from azure.ai.projects.aio import AIProjectClient
from azure.ai.projects.models import PromptAgentDefinition
from azure.identity.aio import AzureCliCredential

from dotenv import load_dotenv
load_dotenv()

foundry_model = os.environ["AZURE_AI_FOUNDRY_MODEL_DEPLOYMENT_NAME"]
instruction = "You are a helpful python programming tutor. Answer questions with code samples and explanation."

async def main():
    async with (
        AzureCliCredential() as credential,
        AIProjectClient(
            endpoint=os.environ["AZURE_AI_PROJECT_ENDPOINT"],
            credential=credential
        ) as project_client,
    ):
        # Create a persistent agent
        created_agent = await project_client.agents.create(
            name="PersistentAgent1",
            definition=PromptAgentDefinition(
                model=foundry_model,
                instructions=instruction,
            ),
        )

        async with ChatAgent(
            chat_client=AzureAIAgentClient(
                async_credential=credential,
                project_client=project_client,
                agent_id=created_agent.id,
            ),
            instructions="You are a helpful assistant."
        ) as agent:
            result = await agent.run("Hello!")
            print(result.text)

asyncio.run(main())

Key Concepts

  1. AIProjectClient: Direct access to the Azure AI Projects API for managing agents at a lower level.

  2. PromptAgentDefinition: Defines the agent’s configuration including the model and instructions.

  3. Agent Creation vs Agent Usage:

    • project_client.agents.create() creates a persistent agent in Azure
    • ChatAgent wraps that agent for conversation

  4. Agent ID: The created agent has a unique ID (created_agent.id) that you can store and reuse in future sessions.

  5. ChatAgent Wrapper: Provides a consistent interface for chatting with the persistent agent.

Persistent Agent Benefits

  • Reusability: Create once, use many times
  • Management: View and manage agents in the Azure portal
  • Consistency: Same configuration across all sessions
  • Team sharing: Multiple users/applications can use the same agent

Best Practices

1. Use Async/Await Consistently

All Foundry agent operations are async. Embrace this for better performance and scalability.

2. Leverage Context Managers

The async with pattern ensures proper resource cleanup, preventing connection leaks.

3. Design Clear Instructions

Your agent’s instructions are crucial. Be specific about:

  • The agent’s role and expertise
  • The tone and format of responses
  • Any constraints or guidelines

4. Type Your Tool Functions

Use type annotations and Pydantic fields for tool parameters. This gives the LLM the context it needs to call your functions correctly.

5. Handle Errors Gracefully

Wrap your agent calls in try/except blocks for production code. Network issues, rate limits, and API errors should be handled gracefully.

6. Consider Streaming for Long Responses

If your agent generates long responses, streaming dramatically improves perceived performance.

Conclusion

The Microsoft Agent Framework makes it remarkably easy to build sophisticated AI agents with Azure AI Foundry. Starting from a basic agent that takes just 15 lines of code, you can progressively add capabilities like streaming, tool calling, code interpretation, and persistence.

The key insight is that agents aren’t just chat interfaces - they’re intelligent systems that can reason about problems and take real actions. By giving them tools, you extend their capabilities beyond language into the real world.
