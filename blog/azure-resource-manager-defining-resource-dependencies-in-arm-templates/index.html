So far in this series, you have learned how to implement basic ARM templates by provisioning storage accounts and virtual networks, made these templates reusable by adding parameters, variables, and expressions, and learned how to reduce complexity of ARM templates using reusable custom user-defined functions. In the last part, you looked at how to secure secrets in an ARM template by provisioning Azure Key Vault and adding & retrieving secrets from the vault. This also completed close to 40% of the template definition for the infrastructure shown in the above diagram. What you have learned so far can be considered as the basics of ARM template development. From today’s part onwards, you will get introduced to advanced techniques in ARM template authoring. First, take a look at creating resource dependencies in ARM templates.

There are multiple ways of defining dependencies among different resources in an ARM template.

DependsOn

You can use the DependsOn element in the resource definition to define resource dependencies. To understand how this works, first see a failure scenario. In this updated template the Microsoft.KeyVault/vaults/secrets resource comes first and the resource this depends on – Microsoft.KeyVault/vaults – comes later in the resource ordering. When you attempt deploying this template, it will fail with an error that the parent resource is not available.

In this example, changing the resource ordering to create the vault first and then attempting to add secrets would solve the problem. To optimize the deployment speed, ARM tries to perform parallel deployment of resources when possible. One way to determine the resource deployment ordering is to calculate the resource dependencies. Since this template has only two resources, ARM simply schedules both resources for deployment. And, this is where you see an issue with the secrets resource failing because the parent resource needed for this isn’t available yet.

Using dependsOn element, you can create explicit dependencies among resources in an ARM template. However, this is not always necessary. Resources in a template can be implicit dependencies as well. This is done using standard template functions such as reference() and list*() functions.

Resource references

Using standard template functions such as reference() and / or any list*() in resource definitions will create implicit dependencies among resources in a template. The reference() function can only be used as a part of the resource properties and template outputs elements.
