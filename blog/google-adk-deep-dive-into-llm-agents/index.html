Introduction to Google Agent Development Kit

In the first article of this series on Google ADK, we looked at a quick introduction to LLM Agents in Google ADK. What we learned so far is just a tiny drop in the ocean. In addition to offering a simple construct (Agent and LlmAgent) for defining agents, ADK provides numerous powerful features and integrations. In today’s article, we will dive into a few advanced concepts related to LlmAgent.

LLM configuration

To control and customize LLM generation, model providers support parameters such as temperature and safety. When using Google ADK, these model configuration parameters are specified using the optional generate_content_config [property](Submodules - Agent Development Kit documentation) of the LlmAgent class. Let us look at an example.

from google.adk.agents.llm_agent import Agent
from google.adk.tools import google_search
from google.genai import types

root_agent = Agent(
    model='gemini-2.5-flash',
    name='code_agent',
    description='Code agent that can generate code in any programming language.',
    instruction='You are a code agent that can generate code in any programming language.',
    tools=[
        google_search
    ],
    generate_content_config= types.GenerateContentConfig(
        temperature=0.7,
        top_p=0.5,
        frequency_penalty=0,
        presence_penalty=0,
    )
)

The generate_content_config requires a value of type types.GenerateContentConfig. You can review a list of all content generation parameters in the Google documentation.
